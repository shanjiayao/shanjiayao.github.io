《Deep Hough Voting for 3D Object Detection in Point Clouds》

  • 作者：Charles R. Qi Or Litany Kaiming He Leonidas J. Guibas
  • 机构：Facebook AI Research && Stanford University
  • 论文水平：ICCV19
  • 关键词：Deep Hough Voting

代码

facebookresearch/votenet

一、背景

与 2D 图像相比，3D 点云具有精确的几何形状和对光照变化的鲁棒性。但是，点云是不规则的。因此，典型的 CNN 不太适合直接用于处理点云数据。

现有的三维目标检测方法

基于2D检测器

  1. 将 Faster 或 Mask R-CNN 等 2D 检测框架扩展到 3D，
  2. 将点云数据投影为规则的 2D 鸟瞰图像，然后应用 2D 检测器定位对象。

缺点：

  1. 2D区域提议则将漏检所有2D检测器漏检的目标
  2. 图像视觉转换需要额外的计算开销
  3. 体素化或者网格化导致信息细节丢失，影响检测精度

基于3D点云处理

  1. 提取点云特征之后，从特征回归候选框的大小和位置

缺点：

  1. 点云只是物体表面的点，可能距离物体中心有一定距离，所以单纯根据提取到的点云特征，并不能很好的回归出准确的Bbox

二、问题

三维检测的表述形式就是Bbox，而Bbox又可以被几个维度的数据表示，分别是中心点、角度、长宽高，所以在自动驾驶领域中，要跟踪的目标尺度默认不变，感知需要完成的任务其实就是回归出中心点以及朝向角，但是，基于点云的网络很难在目标中心附近学习场景的上下文

由于点云数据稀疏性，以及现有的三维激光只能获得物体表面的点云这一特性，在三维目标检测领域中，如何从稀疏且局部的点云获得目标完整的候选框，一直是较为困难的问题。

2D图片做物体检测时，一般一个物体的中心点会在物体内部，中心点周围的像素，一般也都是该物体的像素。在3D点云里面，由于传感器、雷达只能扫描到物体表面的点，而物体中心点一般距离物体表面会有空白区域。对于2D图片，用CNN（RPN）处理一个区域的像素信息从而得出中心点的方法，直觉上是有意义的，中心点很可能就在这个区域的某个像素点上。而对于3D点云，这种方法感觉不是很可行，因为一个物体表面的点距离中心点，有一些距离，中间有很多空白的区域。除非保证CNN的receptive field能够覆盖到物体的表面。receptive field增大后，必然会消耗额外的计算资源，并且三维空间中增大感受野，会引入更多的无用的点，使模型难以训练。

三、本文思想

由于数据的稀疏性，直接从场景点预测边界框参数时面临着一个重大挑战：3D 对象的质心可能远离任何表面点，因此很难用一个步骤准确地回归。为了解决这一问题，作者提出赋予点云深度网络一种类似于经典霍夫投票（Hough voting）的投票机制。通过投票，基本上生成了靠近对象中心的新点，可以对这些点进行分组和聚合（grouped and aggregated），以生成候选框（box proposals），对于每个点，都可以对该点所在物体的中心点进行投票，给出自己的建议。每个点都提出一个中心点的位置，那么，只要所属同一个物体的所有的点中的大部分点的建议不算太差，根据这些大部分点的建议值，就可以大概得出该物体的中心点。

如果大多数点能给出好的投票结果，那么少数服从多数，即可得到中心点。

四、网络结构

1. 传统的霍夫投票

  • 训练过程：
    • 需要根据带注释的对象边界框的图像集合，使用每一个patch与其中心之间的偏移来构建一个映射关系，叫做 Code Book
  • 推断过程：
    • 提取 interest points（SIFT等）
    • 根据 interest points 提取 Image patches
    • 将每一个 Patch 都与 Code Book中的path比较，检索出偏移量，并计算投票
    • 生成Cluster，通过一些聚类算法就可以得到需要检测物体的中心
    • 通过 Cluster ，使用一些分类器和回归器得到需要预测的bounding box。

2. 本文的网络框架

作者提出如何将传统的投票机制用于深度学习中，是一个开放性课题，作者提出了一个框架，分为以下四个部分：

Interest points:

  • Nx3 points → Pointnet++ → Mx(3+C) seeds
  • 输入N个点，经过 Pointnet++， 进行了四层的SA层以及两层FP层，取最后的1024个点作为选出的Seeds，每一个Seed都带有坐标的三个维度特征以及C个其他特征
  • 网络各层参数如下：

Vote:

  • Mx(3+C) seeds → MLP → Mx(3+C) seeds with pos offset and feature offset

  • 通过MLP对对应的(3+C)个通道进行投票，所谓的投票操作就是逐点进行多层感知机的变换，通过有监督的训练，让这一层多层感知机学习参数，进而让得到的(3+C)个维度的特征除了自身的三维坐标以及特性信息外，还额外带有中心点相对于当前点的坐标偏置以及特征偏置

  • 这部分在训练时的label是通过预先计算Box中心到每一个点的距离，相当于每一个点都有一个label，label的值就是距离，然后根据网络预测的值与label计算loss，进而训练网络模型

  • Loss如下：

    • ∆xi：是模型的预测输出
    • ∆xi：*是根据Box提前计算出来的真值
    • Mpos：指在物体表面的点的数量
    • 1[Si on object]：指物体是否在表面，在的话就是1，否则为0，表示这个点不参加loss计算
    • 整体Loss函数其实就是将每一个在表面上的点，其针对偏置的预测和真值之间的欧式距离进行累加，再除以点数

  • 整个投票的过程是用seed的256维度的特征生成259维度的偏置，包括坐标和特征的偏置

  • 判断点在表面上的基准不同数据集不一样，但都是在数据集加载阶段判断的，如果有语义的label，那么直接判断语义的label是否属于当前目标的类别，如果没有，则判断当前点是否在中心点的一定半径范围内

Vote aggregation:

  • Mx(3+C) seeds → SA Module → K x n x (3+C) vote clusters
  • 这一步是通过对点云的采样和聚合来产生K个投票点的聚类结果.
  • sampling : FPS
    • 通过不断计算欧式距离, 产生K个中心点
  • grouping : BallQuery
    • 以K个点为中心, 人工设置半径, 聚类产生K个聚类结果

Object proposals:

  • K x n x (3+C) → Maxpool + MLP → K x Cout → 3D NMS → K boxes
  • 通过前面产生的K个聚类结果, 经过最大池化
